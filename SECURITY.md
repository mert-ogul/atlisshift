# Security Policy

## Supported Versions

We release patches for security vulnerabilities. Which versions are eligible for
receiving such patches depends on the CVSS v3.0 Rating:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

Please report (suspected) security vulnerabilities to **[INSERT EMAIL ADDRESS]**.
You will receive a response within 48 hours. If the issue is confirmed, we will
release a patch as soon as possible depending on complexity but historically
within a few days.

## Responsible Disclosure

We follow responsible disclosure practices:

1. **Do not** open a public GitHub issue for security vulnerabilities
2. **Do** email the security team directly
3. **Do** provide as much detail as possible, including:
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
4. **Do** allow reasonable time for us to address the issue before public disclosure

We appreciate your efforts to keep AtlasShift and its users safe!
