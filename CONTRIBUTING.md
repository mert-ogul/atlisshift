# Contributing to AtlasShift

Thank you for your interest in contributing to AtlasShift!

## Development Setup

1. Fork and clone the repository
2. Install dependencies: `pnpm install`
3. Start development: `pnpm dev`

## Code Style

- We use Biome for linting and formatting
- Run `pnpm format` before committing
- Follow Clean Code principles: small modules, clear naming, no magic constants

## Testing

- Write tests for new features
- Run `pnpm test` before committing
- Aim for good test coverage

## Commits

- Use Conventional Commits format
- Keep commits small (<= 250 LOC)
- Each commit should leave the repo in a green state

## Pull Requests

- Create a feature branch
- Ensure all tests pass
- Update documentation as needed
- Keep PRs focused and reviewable
